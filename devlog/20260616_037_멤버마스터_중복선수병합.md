# 20260616_037 — 멤버 마스터 정규화(중복 선수 병합)

## 배경

"현재 선수들의 union으로 멤버 테이블을 만들고, 팀원 추가 시 거기서 가져오게 해달라"는 요청.
검토 결과 **이미 `Player`(사람) ↔ `TeamMembership`(팀·시즌·등번호) 구조**가 그 목적의
설계였고(CLAUDE.md: 등번호를 Player에 직접 넣지 말 것), admin도 팀 화면에서 기존 Player를
autocomplete로 골라 소속을 추가하게 돼 있었다. 즉 **새 테이블 불필요** — `Player`를 멤버
마스터로 확정하고, 데이터의 중복만 정리하면 된다(사용자 결정: "Player를 멤버 마스터로 정리").

## 문제: 동일인 중복 Player

같은 사람이 K7·40대 두 팀에 등록되며 **별도 Player로 중복 생성**돼 있었다(3명).
- 박대현 `#2`(K7, 등번호2, DF) + `#62`(40대, 빈 레코드, 이벤트 2건)
- 김지훈 `#31`(K7, 84, MF) + `#60`(40대, 빈 레코드, 이벤트 2건)
- 구자헌 `#26`(K7, 52, DF) + `#56`(40대, 빈 레코드, 이벤트 1건)

빈 40대 레코드에 오히려 최근 경기 이벤트가 붙어 있어, 병합 시 멤버십·이벤트를 모두 통합해야 했다.

## 한 일

### `manage.py dedupe_members` (멱등 병합 명령)
- `apps/teams/management/commands/dedupe_members.py`
- 이름이 같은 Player 묶음을 **pk 최소 레코드(정본)로 병합**:
  - 스칼라 필드(birth_year/position/squad/photo/bio) union(정본 빈 칸만 채움)
  - `MatchEvent`·`MatchLineup`·`TeamMembership`을 정본으로 재지정(중복 소속/명단은 정리)
  - 정본·중복의 같은 필드가 **다른 값으로 충돌**하면 그 묶음은 건너뛰고 경고(다른 사람 보호 장치)
  - 중복 삭제. 재실행 시 무변경(멱등). `--dry-run` 지원.

### admin — Player를 멤버 마스터로
- `apps/teams/admin.py` PlayerAdmin: `소속 팀` 컬럼 추가, `memberships__team` 필터,
  "새 팀원은 기존 Player를 골라 추가(중복 생성 금지), 중복은 dedupe_members로 병합" 안내 docstring.

## 검증

- 운영 DB 사본으로 dry-run→실제→재실행(멱등) 확인 후 운영 반영.
- 운영 결과: **Player 62→59명**, TeamMembership 65건 보존(손실 0), 중복 이름 0, 고아 Player 0.
  세 정본이 각각 40대·K7 두 소속과 이벤트를 모두 보유, 포지션 유지.
- admin Player/Team changelist 200, navbar 0.4.3.

## 운영 워크플로우(메모)

- 팀원 추가: 팀 또는 팀 소속 화면에서 **기존 Player를 autocomplete로 선택** → 등번호/시즌 입력.
  명단에 없는 새 사람일 때만 Player 신규 생성.
- 향후 중복이 또 생기면 `python manage.py dedupe_members` 로 정리.
