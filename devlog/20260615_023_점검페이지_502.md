# 20260615_023 — 502 점검 페이지 (배포 중 자동 새로고침)

## 배경
- 새 이미지 배포로 컨테이너 재시작 시 수 초간 502가 발생 → 사용자에게 점검 안내 필요.

## 구현 (nginx, 서버 로컬 — git 비추적)
- 점검 페이지 `/srv/FcSky/maintenance/maintenance.html` (정적, nginx가 직접 서빙).
  - "시스템 점검 중입니다" 안내 + 스피너, `<meta http-equiv="refresh" content="10">`로 **10초 후 자동 새로고침**, 수동 새로고침 링크.
- `/etc/nginx/sites-available/2026biennale` 의 `location /FcSky/` 에
  `error_page 502 503 504 /FcSky/maintenance.html;` 추가 + 내부 전용 alias location.
  - 업스트림(8003) 다운 시 nginx가 점검 페이지를 직접 응답하므로 컨테이너가 죽어도 노출됨.
  - 백업: `2026biennale.bak-maint`.

## 테스트
- 컨테이너 stop → `/FcSky/` 요청이 점검 페이지(메타 refresh 10s) 반환 → start 후 200 복구 확인.
