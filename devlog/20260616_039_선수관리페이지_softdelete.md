# 20260616_039 — 선수(멤버) 관리 페이지 + soft delete

## 한 일

Django admin이 아닌 **사이트 내 전용 Player(멤버 마스터) 관리 페이지**를 신설하고,
navbar에 운영진용 **관리 드롭다운**을 만들어 연결했다. 선수 삭제는 **soft delete**.

### navbar 관리 드롭다운 (`templates/base.html`)
- staff에게만 보이던 단일 "관리"(admin:index) 링크 → **드롭다운**으로 변경:
  - 선수(멤버) 관리 → `teams:player_manage` (사이트 내 페이지)
  - 팀 관리 → `teams:list`
  - 관리자(Django Admin) → `admin:index`

### Player 전체 관리 CRUD (`apps/teams/`)
- URL: `manage/players/` (목록), `add/`, `<pk>/edit/`, `<pk>/delete/`, `<pk>/restore/`.
- 뷰(staff_required):
  - `player_manage` — 전체 선수 목록(이름 검색, 소속 팀 표시). 기본은 활동 선수,
    `?deleted=1` 로 삭제됨 목록(복구용). 활동/삭제됨 건수 탭.
  - `player_create` / `player_master_edit` — `PlayerForm`으로 Player 레코드만 편집
    (팀 소속·등번호는 팀 페이지 담당).
  - `player_master_delete` — **soft delete**(아래).
  - `player_restore` — 복구.
- 템플릿: `player_manage.html`, `player_master_form.html`, `player_master_confirm_delete.html`.

### soft delete (`Player.deleted_at`)
- `Player.deleted_at` 필드 추가(마이그레이션 `0003`). 값이 있으면 삭제로 간주.
- 삭제 시 레코드를 지우지 않고 `deleted_at=now()`만 기록 → **소속·득점·도움·라인업 전부 보존**.
- 숨김 처리:
  - 관리 목록 기본 쿼리에서 제외(삭제됨 탭에서만 노출 + 복구).
  - 팀 상세 로스터(`team_detail`) 쿼리에 `player__deleted_at__isnull=True` 추가.
  - 팀 선수 추가 선택지(`MembershipAddForm`)에서도 제외.
- 복구는 `deleted_at=None`. 소속을 건드리지 않으므로 복구 시 원래 로스터로 그대로 돌아옴.

## 검증

- 운영 DB 사본(마이그레이션 0003 적용) + staff 테스트 클라이언트:
  - 목록/추가/삭제됨 목록 200, 생성(59→60)·수정(포지션/출생) 반영.
  - soft delete → `deleted_at` 설정, 활동 목록에서 제외(상단 플래시 메시지에만 이름 노출),
    삭제됨 목록에 노출, 복구로 원복.
  - 로스터에 있던 선수 soft delete 시 팀 상세에서 사라짐 확인.
