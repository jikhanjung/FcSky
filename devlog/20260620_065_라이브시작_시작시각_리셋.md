# 20260620_065 — 중계 콘솔 'LIVE 시작' 시작시각 리셋 버그 수정

## 증상

실시간 중계 입력 화면에서 한 번 경기를 LIVE 로 시작하면 `live_started_at`(자동 시계 기준점)이
기록되는데, 이후 **경기를 종료하고 이벤트·스코어를 모두 지워 '예정'으로 되돌려도** 시작시각이
그대로 남아 있었다. 그래서 같은 경기를 **다시 LIVE 로 시작하면 처음 입력된 시작시각부터**
카운트되어 경기 시간이 비정상적으로 길게 표시됐다.

## 원인

`apps/matches/views.py` 의 `_handle_live_action()` `start` 액션:

```python
if match.live_started_at is None:        # 기록돼 있으면 무조건 보존
    match.live_started_at = timezone.now()
```

`is None` 가드는 원래 "종료 후 재개 시 시계 연속성 유지" 의도였으나, 예정 상태로 되돌린 새 경기에도
적용되어 옛 시각이 보존됐다.

## 수정

시작 직전 상태로 두 경우를 구분:

- **예정(SCHEDULED) → 시작**: 새 경기로 보고 `live_started_at` 을 **현재 시각으로 새로 세팅**.
- **종료(FINISHED) → 재개**: 시계 연속성을 위해 기존 시각 **보존**.

```python
fresh_start = match.status == Match.Status.SCHEDULED
match.status = Match.Status.LIVE
if fresh_start or match.live_started_at is None:
    match.live_started_at = timezone.now()
    fields.append("live_started_at")
```

'예정으로 되돌리기'는 admin 에서 상태 변경으로 이뤄지므로(`live_started_at` 은 남음), 이 수정으로
**다음 시작 때 시작시각이 자동으로 갱신**되어 사용자가 별도로 지울 필요가 없다.

## 테스트

`apps/matches/tests.py` 에 `LiveStartTest` 추가(`_handle_live_action` 직접 단위 테스트):

- `test_fresh_start_resets_time` — 예정 상태 + 옛 시각 → 시작 시 새 시각으로 갱신.
- `test_resume_preserves_time` — 종료 상태 + 시각 → 재개 시 기존 시각 보존.
- `test_first_start_sets_time` — 시각 없음 → 현재 시각 세팅.

> 검증은 테스트 서버에서 수행(dolfinid 배포 호스트엔 Django venv 없음).
