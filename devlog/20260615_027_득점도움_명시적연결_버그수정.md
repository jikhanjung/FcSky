# 20260615_027 — 득점-도움 명시적 연결 (분 미입력 시 오결합 버그 수정)

## 증상
- 분(minute)을 입력하지 않은 채 일부 득점에만 도움을 넣고 저장하면,
  도움이 엉뚱한 득점에 붙고, 재편집 시 모든 득점 행의 도움이 "첫 도움 준 사람"으로 표시됨.

## 원인
- 득점↔도움을 **(팀, 분) 휴리스틱**으로 짝지었는데 분이 모두 `None`이면 구분이 불가.
  - 폼 초기값: `filter(side, minute=None).first()` → 항상 첫 도움 반환.
  - 저장 재동기화: 같은 (side, None)으로 재생성 → 어느 득점 도움인지 정보 소실.

## 수정 — 명시적 링크
- `MatchEvent.goal` (self FK, `related_name="assists"`) 추가 — 도움이 자기 득점을 직접 가리킴.
  마이그레이션 `matches.0003_matchevent_goal`.
- `forms.py` — 득점 행 도움 초기값을 `inst.assists.first()`(링크)로.
- `views.py` `match_edit` — 도움 재생성 시 `goal=goal`로 링크 저장.
- `views.py` `_build_timeline` — 도움 페어링을 링크(goal_id) 기준으로. 링크 없는 과거
  데이터는 같은 팀 도움을 **순서대로** 소비하는 폴백(기존 표시 유지).

## 테스트
- 득점 4개(1·2 도움없음, 3→A, 4→B, 분 미입력) 저장 → 득점별 도움 정확히 연결,
  재편집 행별 도움 선택 정확. 레거시 match(분 없는 기존 도움 3건)도 타임라인 3쌍 정상 표시.

## 비고
- 기존에 입력해둔 경기는 도움이 링크 없이 저장돼 있어, 한 번 **결과 편집에서 다시 저장**하면
  도움이 정확한 득점에 연결됨(이후 영구 정확).
