# 20260615_005 — Docker 이미지 (honestjung/fcsky 0.1.0)

## 목표
애플리케이션을 컨테이너 이미지로 패키징. 이미지명 `honestjung/fcsky`, 버전 0.1.0 시작.
Docker 관련 파일은 `deploy/` 디렉토리에 모은다.

## 설정 변경 (컨테이너 대응)
- `config/settings.py`: 환경변수로 재정의 가능하게.
  - `SECRET_KEY`, `DEBUG`(기본 true=개발), `ALLOWED_HOSTS`(콤마 구분,
    기본 localhost/127.0.0.1/0.0.0.0), `CSRF_TRUSTED_ORIGINS` ← 환경변수.
  - WhiteNoise 미들웨어 추가(SecurityMiddleware 직후) + `STORAGES.staticfiles`
    = `CompressedManifestStaticFilesStorage` → DEBUG=False에서도 정적 파일 서빙.
- `requirements.txt`: `gunicorn==23.0.0`, `whitenoise==6.8.2` 추가.

## deploy/ 구성
- `deploy/Dockerfile` — python:3.12-slim, 의존성 캐시 레이어, collectstatic,
  비루트(appuser) 실행, gunicorn(3 worker) 기동. 빌드 컨텍스트는 프로젝트 루트.
- `deploy/docker-entrypoint.sh` — 시작 시 `migrate`, 선택적 `seed`(DJANGO_SEED=1),
  선택적 관리자 생성(DJANGO_SUPERUSER_*) 후 CMD 실행.
- `deploy/Dockerfile.dockerignore` — BuildKit이 Dockerfile 옆 파일을 우선 사용.
  db.sqlite3·staticfiles·media·data·devlog·.git 등 제외.
- `deploy/docker-compose.yml`, `deploy/README.md` — 실행 안내/예시.

## 빌드 / 검증
```bash
docker build -f deploy/Dockerfile -t honestjung/fcsky:0.1.0 -t honestjung/fcsky:latest .
```
- 이미지 284MB. collectstatic 128개(382 post-process) 성공.
- 컨테이너 기동 테스트(포트 8001, DJANGO_SEED=1):
  migrate→seed(팀3·선수39·경기6·이벤트17)→gunicorn 정상.
  `/`·`/matches/`·`/matches/scorers/` 200, `/admin/` 302,
  `/static/img/fcskylogo.png` 200(WhiteNoise) 확인.

## 비고

  PostgreSQL 전환 권장.
- 로컬 개발 동작은 그대로(DEBUG 기본 true). venv에도 gunicorn/whitenoise 설치됨.
